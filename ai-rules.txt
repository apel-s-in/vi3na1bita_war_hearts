АРХИТЕКТУРА:

- vi3na1bita_war_hearts — отдельная игра для Game Center vi3na1bita-games.
- Публичный путь после деплоя: /Games/war_hearts/.
- Не создавать папку Games/ внутри этого репозитория.
- Корень репозитория деплоится в Yandex Object Storage prefix /Games/war_hearts/.
- Общий сетевой модуль подключается из /Games/common/network-bridge.js.
- Игра не должна читать OAuth token, parent localStorage, parent IndexedDB или parent.window.
- Связь с Game Center допускается только через postMessage или безопасные параметры запуска.
- Игра может иметь собственный WebRTC DataChannel, игровой чат и голосовой PTT внутри игровой сессии.
- Голос включается только по пользовательскому действию.
- На раннем этапе не добавлять service worker.
- Основная ориентация интерфейса: мобильная вертикальная 9:16.
- Игра должна работать standalone-preview, даже если Game Center или network bridge недоступны.
- Любые таймеры и polling должны останавливаться или деградировать при document.hidden.

ИГРОВЫЕ ПРАВИЛА "ВОЙНА СЕРДЕЦ":

- Основа правил — классический морской бой 10×10.
- Расстановка кораблей уже считается правильной: корабли строго горизонтально/вертикально, между кораблями минимум 1 клетка зазора по всем сторонам и диагоналям.
- Ход переходит сопернику только при промахе.
- Если игрок попал, ранил или полностью уничтожил корабль, он продолжает стрелять до первого промаха.
- Каждый выстрел игрока должен подтверждаться красивой модалкой: координата в формате буква+цифра, кнопки "Выстрел" и "Отмена".
- При выборе клетки перед подтверждением на поле соперника показываются пунктирные линии по горизонтали/вертикали и прицел в выбранной клетке.
- После полного уничтожения корабля вокруг него автоматически отмечаются гарантированно пустые клетки: серые неактивные клетки с белым диагональным крестом.
- Автоматическая зона вокруг убитого корабля не должна затирать реальные выстрелы: попадания и промахи всегда остаются видимыми.
- Компьютер не должен стрелять в уже открытые клетки и в клетки, которые помечены как гарантированно пустые после уничтожения корабля.
- В чате должна сохраняться история выстрелов: кто стрелял, координата, результат — промах, ранил, убил.
- AI targeting должен жить в src/game/targeting.js.
- Словарь достижений должен жить в src/game/achievements.js.
- История боёв должна храниться через GC_SAVE_DATA ключом matchHistory с лимитом последних 50 боёв.
- После завершения любого боя должны раскрываться обе расстановки кораблей: игрока и соперника/компьютера.
- Для сетевого P2P-боя использовать commit/reveal: до боя BOARD_COMMIT, после боя BOARD_REVEAL.
- Клиент не должен получать полную расстановку соперника до финального BOARD_REVEAL.
- После BOARD_REVEAL клиент должен проверять hash commit, валидность расстановки и соответствие результатов выстрелов раскрытой доске.
- Fair-play helpers должны жить в src/game/fair-play.js.
- Проверка соответствия сетевых SHOT_RESULT финальному BOARD_REVEAL должна жить в src/game/shot-verifier.js.
- В сетевой игре на экранах приглашения, расстановки и боя должна всегда показываться жёлтая строка network-индикатора с кратким состоянием P2P-соединения и ожиданий.
- Проверка очередности сетевых ходов должна жить в src/game/turn-guard.js.
- Сетевой клиент должен отклонять SHOT не в ход соперника, SHOT_RESULT без ожидаемого SHOT, устаревший shotId и дубли выстрелов.
- Network heartbeat и soft-timeout должны жить в src/game/network-watchdog.js.
- Network watchdog должен деградировать или останавливаться при document.hidden.
- Реванш с компьютером и preview-соперником не должен сразу запускать бой: сначала экран расстановки, затем розыгрыш камень-ножницы-бумага.
- Сетевой реванш должен идти только через REMATCH_REQUEST / REMATCH_ACCEPT / REMATCH_REJECT, затем новая расстановка, READY/BOARD_COMMIT и только потом RPS.
- Экран выбора соперника не должен открывать пустой battle preview без активного матча.
- Дублирующую логику старта локального боя нужно выносить в общий helper, чтобы не расходились правила старта, draft, fair-play и timers.
- Любой финал матча должен быть защищён от повторного выполнения.
